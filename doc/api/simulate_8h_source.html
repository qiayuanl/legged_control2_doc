35// The viewer itself doesn't require a reentrant mutex, however we use it in
36// order to provide a Python sync API that doesn't require separate locking
37// (since sync is by far the most common operation), but that also won't
38// deadlock if called when a lock is already held by the user script on the
39// same thread.
40class SimulateMutex : public std::recursive_mutex {};
41using MutexLock = std::unique_lock<std::recursive_mutex>;
